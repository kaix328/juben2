# 🎉 自动关系分析功能已添加

## ✅ 问题解决

**之前的问题**: 关系图谱显示 "19 个节点 0 个关系"  
**原因**: 关系数据需要手动添加或自动生成  
**解决方案**: 实现了智能的自动关系分析功能

---

## 🚀 新增功能

### 1. 自动关系分析按钮 ✨

在所有 4 个标签页的工具栏中添加了：

```
[关系图谱 (显示关系数量)] [自动分析关系 ✨]
```

### 2. 智能关系分析算法

系统会自动分析以下关系：

#### 📍 角色 ↔ 场景关系
- **分析依据**: 共同标签
- **关系类型**: `appears_in` (出现在)
- **强度计算**: 0.3 + 共同标签数 × 0.2 (最高 1.0)

#### 🎯 角色 ↔ 道具关系
- **分析依据**: 
  - 角色描述中提到道具名称 (强度 0.6)
  - 共同标签 (强度 0.4)
- **关系类型**: `uses` (使用)

#### 👔 角色 ↔ 服装关系
- **分析依据**:
  - 服装的 `characterId` 字段 (强度 1.0)
  - 角色描述中提到服装名称 (强度 0.7)
- **关系类型**: `wears` (穿着)

#### 👥 角色 ↔ 角色关系
基于描述中的关键词自动识别：

| 关系类型 | 关键词 | 强度 |
|---------|--------|------|
| 朋友 (friend) | "朋友" | 0.8 |
| 敌人 (enemy) | "敌人"、"对手" | 0.8 |
| 家人 (family) | "家人"、"父亲"、"母亲"、"兄弟"、"姐妹" | 0.9 |
| 恋人 (lover) | "恋人"、"爱人" | 0.9 |
| 同事 (colleague) | "同事"、"同僚" | 0.7 |
| 师徒 (master_student) | "师父"、"徒弟"、"老师"、"学生" | 0.8 |
| 相关 (related_to) | 多个共同标签 | 0.3-0.7 |

#### 🏞️ 场景 ↔ 道具关系
- **分析依据**: 场景描述中提到道具名称
- **关系类型**: `related_to` (相关)
- **强度**: 0.6

## 📖 使用方法

### 步骤 1: 点击"自动分析关系"按钮

在任意标签页的工具栏中，点击带有 ✨ 图标的"自动分析关系"按钮。

### 步骤 2: 等待分析完成

系统会自动：
1. 扫描所有资产（角色、场景、道具、服装）
2. 分析名称、描述、标签
3. 识别关系关键词
4. 计算关系强度
5. 生成关系数据

### 步骤 3: 查看结果

分析完成后会显示提示：
- ✅ "已自动分析并添加 X 个关系"
- ℹ️ "未发现新的关系"（如果已经分析过）

### 步骤 4: 打开关系图谱

点击"关系图谱"按钮，现在应该能看到：
- 节点之间的连线（关系）
- 不同颜色的连线（不同关系类型）
- 不同粗细的连线（关系强度）

## 🎨 关系可视化

### 节点形状和颜色
- 🔵 **圆形** = 角色 (蓝色)
- 🟢 **方形** = 场景 (绿色)
- 🟠 **三角形** = 道具 (橙色)
- 🟣 **菱形** = 服装 (紫色)

### 关系线条颜色
- 🟢 **绿色** = appears_in (出现在)
- 🟠 **橙色** = uses (使用)
- 🟣 **紫色** = wears (穿着)
- 🔵 **蓝色** = friend (朋友)
- 🔴 **红色** = enemy (敌人)
- 🩷 **粉色** = family (家人)
- ❤️ **深红** = lover (恋人)
- 💙 **靛蓝** = colleague (同事)
- 🩵 **青色** = master_student (师徒)
- ⚫ **灰色** = related_to (相关)

### 线条粗细
- **细线** = 弱关系 (强度 0.3-0.5)
- **中线** = 中等关系 (强度 0.5-0.7)
- **粗线** = 强关系 (强度 0.7-1.0)

## 💡 优化建议

### 提高关系识别准确度

#### 1. 完善资产描述
在角色、场景、道具、服装的描述中：
- ✅ 明确提到相关资产的名称
- ✅ 使用关系关键词（朋友、敌人、使用等）
- ✅ 描述具体的互动和联系

**示例**:
```
角色描述：
"李明是一名侦探，经常使用放大镜调查案件。
他的好朋友王华是警察，两人经常合作破案。"

这样会自动识别：
- 李明 → 放大镜 (uses)
- 李明 → 王华 (friend)
```

#### 2. 使用标签分类
为相关资产添加相同的标签：
- ✅ 场景标签：如 "办公室"、"夜晚"、"紧张"
- ✅ 角色标签：如 "主角"、"反派"、"办公室"
- ✅ 道具标签：如 "武器"、"办公室"
- ✅ 服装标签：如 "正装"、"现代"

**示例**:
```
角色：李明 - 标签：[主角, 侦探, 办公室]
场景：侦探事务所 - 标签：[办公室, 白天]
道具：放大镜 - 标签：[侦探, 办公室]

自动识别：
- 李明 → 侦探事务所 (appears_in) - 共同标签：办公室
- 李明 → 放大镜 (uses) - 共同标签：侦探、办公室
```

#### 3. 设置服装关联
在服装数据中设置 `characterId` 字段：
```typescript
costume.characterId = character.id
```
这样会自动建立强关系 (强度 1.0)

## 🔄 重复分析

### 智能去重
- ✅ 系统会自动检测已存在的关系
- ✅ 不会重复添加相同的关系
- ✅ 可以多次点击"自动分析关系"按钮
- ✅ 只会添加新发现的关系

### 更新关系
如果修改了资产的描述或标签：
1. 再次点击"自动分析关系"
2. 系统会发现新的关系
3. 自动添加到图谱中

## 🐛 故障排除

### 问题 1: 点击后显示"未发现新的关系"

**可能原因**:
- 已经分析过，没有新关系
- 资产描述太简单
- 没有使用关系关键词
- 没有共同标签

**解决方案**:
1. 完善资产描述
2. 添加关系关键词
3. 为相关资产添加共同标签

### 问题 2: 关系不准确

**可能原因**:
- 描述中偶然提到其他资产名称
- 关键词误匹配

**解决方案**:
- 💡 未来版本会支持手动删除关系
- 暂时可以通过修改描述避免误匹配

### 问题 3: 关系太少

**可能原因**:
- 资产之间缺少明确联系
- 描述不够详细

**解决方案**:
1. 在描述中明确提到相关资产
2. 使用关系关键词
3. 添加更多标签
