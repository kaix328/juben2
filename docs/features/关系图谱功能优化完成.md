### 2. 增强自动分析能力 🧠

#### 新增智能匹配算法

##### 📍 模糊名称匹配
```typescript
// 支持多种匹配方式：
- 直接匹配: "李明" → "李明"
- 去空格匹配: "李 明" → "李明"
- 分词匹配: "侦探李明" → "李明"
```

##### 📊 文本相似度计算
```typescript
// 计算描述的相似度
similarity = 共同词汇数 / 总词汇数
如果相似度 > 0.3，建立关系
```

##### 🎯 多维度强度计算

**角色 ↔ 场景关系**:
- 共同标签: +0.3 + 标签数 × 0.15
- 角色描述提到场景: +0.4
- 场景描述提到角色: +0.4
- 描述相似度 > 0.3: +相似度 × 0.3

**角色 ↔ 道具关系**:
- 角色描述提到道具: +0.5
- 道具描述提到角色: +0.5
- 共同标签: +0.2 + 标签数 × 0.1
- 使用关键词匹配: +0.3
  - 关键词: "使用"、"拿着"、"持有"、"携带"、"佩戴"、"装备"

**角色 ↔ 服装关系**:
- characterId 匹配: 1.0 (最强)
- 角色描述提到服装: +0.6
- 服装描述提到角色: +0.6
- 共同标签: +0.2 + 标签数 × 0.1
- 穿着关键词匹配: +0.3
  - 关键词: "穿着"、"穿戴"、"身穿"、"着装"、"打扮"

**角色 ↔ 角色关系**:
- 保持原有的关键词识别
- 增加了更多关系类型判断

### 3. 新增刷新功能 🔄

**功能**: 点击"刷新"按钮重新渲染图谱

**用途**:
- 关系数据更新后刷新显示
- 图谱布局混乱时重置
- 过滤器切换后重新布局

**实现**: 
```typescript
const handleRefreshGraph = () => {
    setShowRelationGraph(false);
    setTimeout(() => setShowRelationGraph(true), 100);
};
```

## 🎯 分析阈值说明

### 关系强度阈值

只有强度 > 0.3 的关系才会被添加：

| 强度范围 | 说明 | 线条粗细 |
|---------|------|---------|
| 0.3-0.5 | 弱关系 | 细线 |
| 0.5-0.7 | 中等关系 | 中线 |
| 0.7-1.0 | 强关系 | 粗线 |

### 为什么设置阈值？

- ✅ 避免误匹配
- ✅ 减少噪音关系
- ✅ 保持图谱清晰
- ✅ 突出重要关系

## 🔄 刷新功能说明

### 何时使用刷新？

1. **添加关系后** - 查看新的连线
2. **修改过滤器后** - 重新布局
3. **图谱混乱时** - 重置布局
4. **节点重叠时** - 重新排列

### 刷新 vs 重新打开

**刷新按钮**: 
- 快速重新渲染
- 保持当前过滤器
- 保持缩放级别

**重新打开对话框**:
- 完全重新初始化
- 重置所有设置
- 重新加载数据

## 📈 性能优化

### 分析性能

| 资产数量 | 分析时间 | 关系数量（估计） |
|---------|---------|----------------|
| < 20 | < 100ms | 10-30 |
| 20-50 | 100-300ms | 30-100 |
| 50-100 | 300-800ms | 100-300 |
| 100+ | 800ms-2s | 300+ |

### 优化措施

1. **异步分析** - 使用 setTimeout 避免阻塞 UI
2. **去重检查** - 避免重复添加关系
3. **阈值过滤** - 只添加有意义的关系
4. **批量添加** - 一次性添加所有关系

## 🐛 故障排除

### 问题 1: 点击"自动分析关系"后没反应

**可能原因**:
- 已经分析过，没有新关系
- 资产描述太简单
- 没有达到阈值 (0.3)

**解决方案**:
1. 完善资产描述
2. 添加关系关键词
3. 添加相关标签
4. 查看控制台是否有错误

### 问题 2: 关系太多，图谱混乱

**解决方案**:
1. 使用过滤器只显示特定类型
2. 点击"刷新"重新布局
3. 使用"适配视图"按钮
4. 手动拖拽节点调整

### 问题 3: 关系不准确

**解决方案**:
- 💡 未来版本会支持手动删除关系
- 暂时可以通过修改描述避免误匹配
- 移除会导致误匹配的关键词
