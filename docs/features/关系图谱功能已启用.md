2. 查看自动生成的关系网络

### 步骤 4：测试交互功能
- **拖拽节点**：点击并拖动任意节点
- **缩放视图**：使用工具栏的放大/缩小按钮，或鼠标滚轮
- **过滤类型**：使用下拉菜单只显示特定类型的资产
- **查看详情**：点击节点查看侧边栏详情
- **自动适配**：点击 "适配视图" 按钮

### 步骤 5：理解图谱元素
- **圆形节点** = 角色（蓝色）
- **方形节点** = 场景（绿色）
- **三角形节点** = 道具（橙色）
- **菱形节点** = 服装（紫色）
- **箭头** = 关系方向
- **线条粗细** = 关系强度

## 🔧 技术细节

### 依赖库
- **vis-network**: 强大的网络可视化库
  - 支持大规模图谱（1000+ 节点）
  - 丰富的交互功能
  - 高性能渲染
  
- **vis-data**: 响应式数据集
  - 自动更新视图
  - 高效的数据操作
  - 支持过滤和排序

### 性能优化
- **按需渲染**：只渲染可见区域
- **事件节流**：防止频繁重绘
- **内存管理**：组件卸载时清理资源
- **懒加载**：对话框打开时才初始化

### 数据流
```
useRelationGraph Hook
  ↓
生成节点和边数据
  ↓
RelationGraphDialog 组件
  ↓
vis-network 渲染
  ↓
用户交互
  ↓
回调事件
```

## 🐛 故障排除

### 如果图谱不显示
1. 检查浏览器控制台是否有错误
2. 确认 vis-network 已正确安装
3. 刷新页面重新加载

### 如果性能较慢
1. 减少节点数量（使用过滤器）
2. 禁用物理引擎（在代码中设置 `physics.enabled: false`）
3. 降低稳定化迭代次数

### 如果布局混乱
1. 点击 "适配视图" 按钮
2. 手动拖拽节点调整
3. 切换过滤器重新布局
