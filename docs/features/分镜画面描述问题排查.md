## 🎯 问题根源

### 主要问题：AI 返回的 description 可能为空

当使用 AI 提取分镜时，如果 AI 模型返回的 JSON 中 `description` 字段为空或缺失，代码会设置为空字符串，导致画面描述为空。

**可能的原因**:
1. AI 提示词中没有强调 `description` 字段的重要性
2. AI 模型理解偏差，将描述信息放在了其他字段
3. 场景信息不足，AI 无法生成有效描述

---

## ✅ 解决方案

### 方案 1: 增强 AI 提示词（推荐）

在 `extractStoryboard` 函数的提示词中，强调 `description` 字段：

```typescript
const prompt = `你是专业分镜设计师...

【JSON 格式】
[{
    "sceneId":"场景ID",
    "description":"🔴 必填！详细的画面描述，包含：场景、角色、动作、表情、光影氛围、视觉隐喻（至少50字）",
    "shotSize":"MS",
    // ... 其他字段
}]
`;
```

### 方案 2: 添加智能 Fallback（推荐）

在映射 AI 返回数据时，如果 `description` 为空，自动生成：

```typescript
const allPanels = shots.map((shot: any, index: number) => {
    const matchedScene = sceneMap.get(shot.sceneId) || scenes[Math.floor(index / 3)];
    
    // 🆕 智能生成 description
    let description = shot.description || '';
    if (!description || description.length < 10) {
        // 根据其他字段生成描述
        const shotCN = SHOT_CODE_TO_CN[shot.shotSize || 'MS'] || '中景';
        const location = matchedScene?.location || '场景';
        const timeOfDay = matchedScene?.timeOfDay || '';
        const characters = shot.characters?.join('、') || '角色';
        const action = shot.dialogue ? '说话' : matchedScene?.action?.substring(0, 50) || '在场景中';
        
        description = `${shotCN}，${location}${timeOfDay ? `，${timeOfDay}` : ''}。${characters}${action}`;
    }
    
    return {
        // ...
        description: description,
        // ...
    };
});
```

### 方案 3: 使用 smartFillPanel 补充

`smartFillPanel` 函数已经有逻辑补充空字段，但可能需要增强：

```typescript
// 在 smartFillPanel 中添加
if (!panel.description || panel.description.length < 10) {
    const shotCN = panel.shot || '中景';
    const location = matchedScene?.location || '场景';
    const characters = panel.characters?.join('、') || '角色';
    const action = panel.dialogue ? '说话，表情变化' : '在场景中活动';
    
    panel.description = `${shotCN}，${location}。${characters}${action}`;
}
```
