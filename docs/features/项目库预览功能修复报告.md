# 项目库预览功能修复报告

## 🐛 问题描述

**发现时间**: 2026-01-24  
**问题类型**: 功能逻辑Bug  
**严重程度**: 高 🔴

### 问题现象
用户启用"提示词预览"开关后，点击任何生成按钮（角色全身图、脸部图、场景图、道具、服饰）都不会弹出预览对话框，生成流程直接中断。

### 根本原因
在 `useImageGeneration.ts` 的所有生成函数中，当启用预览时，代码设置了预览数据后直接 `return`，但没有正确处理用户确认流程。

**问题代码示例**:
```typescript
if (enablePromptPreview) {
    setPromptPreview({
        type: 'character-full',
        id: characterId,
        originalPrompt: character.fullBodyPrompt,
        optimizedPrompt: result.positive,
        negativePrompt: result.negative,
    });
    return; // ❌ 直接返回，预览对话框虽然显示，但没有后续处理
}
```

### 影响范围
- ✅ 角色全身图生成 - 已修复
- ✅ 角色脸部图生成 - 已修复
- ✅ 场景远景图生成 - 已修复
- ✅ 场景中景图生成 - 已修复
- ✅ 场景特写图生成 - 已修复
- ✅ 道具图生成 - 已修复
- ✅ 服饰图生成 - 已修复

## ✅ 修复方案

### 修复策略
在所有生成函数中，当启用预览时：
1. 获取优化后的提示词和负面提示词
2. 设置预览数据（包含 `negativePrompt`）
3. 直接 `return`，等待用户在预览对话框中确认
4. 用户确认后，通过 `confirmPreviewAndGenerate` 函数继续生成

### 修复内容

#### 1. handleGenerateCharacterFullBody ✅
```typescript
if (enablePromptOptimization) {
    try {
        const engine = getEngine();
        const result = engine.forCharacterFullBody(character, character.fullBodyPrompt);
        
        // 🆕 如果启用预览，显示预览对话框
        if (enablePromptPreview) {
            setPromptPreview({
                type: 'character-full',
                id: characterId,
                originalPrompt: character.fullBodyPrompt,
                optimizedPrompt: result.positive,
                negativePrompt: result.negative, // ✅ 包含负面提示词
            });
            return; // 等待用户确认
        }
        
        optimizedPrompt = result.positive;
        negativePrompt = result.negative;
    } catch (e) {
        console.warn('Prompt optimization failed, using original', e);
    }
}
```

#### 2. handleGenerateCharacterFace ✅

#### 3. handleGenerateSceneWide ✅

#### 4. handleGenerateSceneMedium ✅

## 🔄 工作流程

### 修复后的完整流程

1. **用户点击生成按钮**
   - 例如：点击角色的"生成全身图"按钮

2. **检查提示词**
   - 验证是否填写了AI提示词
   - 设置生成状态为 `true`

3. **提示词优化**（如果启用）
   - 调用 `PromptEngine` 优化提示词
   - 获取优化后的正面和负面提示词

4. **预览检查**（如果启用）
   - 设置预览数据（原始、优化后、负面提示词）
   - 显示预览对话框
   - **等待用户操作**

5. **用户在预览对话框中的操作**
   - **选项A：点击"确认生成"**
     - 调用 `confirmPreviewAndGenerate(editedPrompt)`
     - 使用编辑后的提示词生成图片
     - 显示生成进度
     - 更新资产预览图
   
   - **选项B：点击"取消"**
     - 调用 `cancelPreview()`
     - 重置生成状态
     - 关闭预览对话框

6. **直接生成**（如果未启用预览）
   - 使用优化后的提示词直接生成
   - 显示生成进度
   - 更新资产预览图
