**创建时间**: 2026-01-19  
**状态**: 已添加调试日志，待测试

## 📋 问题描述

用户在章节编辑器中点击"分析文中资产"按钮后，智能助手面板只显示了识别到的角色，但没有显示场景和道具。

### 预期行为
- ✅ 显示识别到的角色
- ❌ 显示识别到的场景
- ❌ 显示识别到的道具

## 🔍 可能的原因分析

### 原因 1: AI 提取失败

**症状**: 
- 控制台显示 `scenes: 0, props: 0`
- 只有 `characters` 有数据

**原因**:
- AI 模型没有正确识别场景和道具
- 文本内容不够详细
- Prompt 提示词需要优化

**解决方案**:
1. 检查 `src/app/utils/ai/assetExtractor.ts` 中的 prompt
2. 确保文本包含明确的场景和道具描述
3. 测试不同的文本内容

### 原因 2: 数据转换问题

**症状**:
- 控制台显示 `scenes: 2, props: 1`（有数据）
- 但转换后的预览数据只有角色

**原因**:
- 数据转换逻辑有问题
- `result.scenes` 或 `result.props` 的数据结构不符合预期

**解决方案**:
1. 检查 `result.scenes` 和 `result.props` 的数据结构
2. 确保 `map` 函数正确处理数据
3. 检查是否有空数组或 undefined

### 原因 3: UI 显示问题

**症状**:
- 控制台显示转换后有 6 个资产
- 但 UI 只显示角色

**原因**:
- `AnalysisSidebar` 组件的渲染逻辑有问题
- 场景和道具的 Badge 颜色或样式导致不可见

**解决方案**:
1. 检查 `src/app/pages/ChapterEditor/components/AnalysisSidebar.tsx`
2. 确保所有类型的资产都能正确渲染
3. 检查 CSS 样式是否有问题

## 🐛 已知问题

### 问题 1: AI 提取不稳定

**描述**: 相同的文本，多次分析可能得到不同的结果

**影响**: 中等

**状态**: 已知，AI 模型的固有特性

**解决方案**: 
- 优化 prompt，使其更加明确
- 添加重试机制
- 提供手动添加功能

### 问题 2: 场景识别率低

**描述**: AI 经常无法识别隐含的场景

**影响**: 高

**状态**: 待优化

**解决方案**:
- 改进 prompt，增加场景识别的权重
- 添加场景关键词提示
- 提供场景模板

### 问题 3: 道具识别过于宽泛

**描述**: AI 可能将普通物品也识别为道具

**影响**: 低

**状态**: 可接受

**解决方案**:
- 添加道具重要性评分
- 允许用户删除不需要的道具
- 提供道具筛选功能
