# 模块导入错误修复指南

## 问题描述
错误：`Failed to fetch dynamically imported module`

## 原因
这是 Vite 开发服务器的常见问题，通常由以下原因引起：
1. 端口变更（5173 → 5174）
2. 浏览器缓存了旧的模块引用
3. 热更新（HMR）缓存问题

## 解决方案

### 方法 1：清除浏览器缓存并刷新（推荐）

1. **在浏览器中按 `Ctrl + Shift + R`**（Windows）或 `Cmd + Shift + R`（Mac）
   - 这会强制刷新并清除缓存

2. **或者手动清除缓存：**
   - 打开开发者工具（F12）
   - 右键点击刷新按钮
   - 选择"清空缓存并硬性重新加载"

### 方法 2：更新 URL

当前服务器运行在新端口：
- ❌ 旧地址：`http://localhost:5173`
- ✅ 新地址：`http://localhost:5174`

**请访问：** `http://localhost:5174/#/projects/1769012057420-pgjay5np7/assets?tab=scene`

### 方法 3：重启开发服务器

如果上述方法无效，请执行：

```bash
# 停止当前服务器（Ctrl + C）
# 然后重新启动
npm run dev
```

### 方法 4：清除 Vite 缓存

```bash
# 删除 Vite 缓存目录
rm -rf node_modules/.vite

# 重新启动
npm run dev
```

## 验证修复

访问新地址后，检查：
1. ✅ 页面正常加载
2. ✅ 没有模块导入错误
3. ✅ 所有功能正常工作

## 注意事项

- 开发服务器端口可能会变化（如果 5173 被占用）
- 始终使用浏览器显示的实际端口
- 定期清除浏览器缓存以避免此类问题
