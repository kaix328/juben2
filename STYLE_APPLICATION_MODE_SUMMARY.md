## 🎯 核心功能

### 1. **智能保护机制**

#### 保护手动编辑的提示词
```typescript
// 旧系统：无条件覆盖所有提示词
batchApplyStyleToCharacters(characters, style);

// 新系统：智能跳过已编辑的提示词
if (!styleSettings.protectManualEdits || !char.fullBodyPrompt?.trim()) {
  // 只应用到未设置或空的提示词
  updates.fullBodyPrompt = engine.forCharacterFullBody(char).positive;
}
```

**效果：**
- ✅ 防止意外覆盖精心调整的提示词
- ✅ 新建资源或空提示词会被自动应用
- ✅ 用户可以选择关闭保护（覆盖全部）

## 🔄 工作流程

### 场景1：批量应用导演风格（推荐设置）

```
用户点击"应用导演风格到全部"
           ↓
检查 confirmBeforeApply 设置
           ↓
   [开启] 显示确认对话框
           ├─ 影响范围统计
           ├─ 详细清单
           └─ 预览对比（如果showPreview开启）
           ↓
       用户确认
           ↓
检查 protectManualEdits 设置
           ↓
   [开启] 跳过已有提示词
   [关闭] 覆盖所有提示词
           ↓
      应用完成
           ↓
     显示成功提示
```

### 场景2：新建资源（autoApplyToNew 开启）

```
用户创建新角色
       ↓
检查 autoApplyToNew 设置
       ↓
  [开启] 自动应用导演风格
  [关闭] 提示词为空
       ↓
   保存角色
```

### 场景3：手动模式 vs 自动模式

#### 手动模式（mode: 'manual'）
```
用户修改导演风格
       ↓
   保存成功
       ↓
提示："导演风格已保存"
       ↓
需要手动点击"应用导演风格到全部"
```

#### 自动模式（mode: 'auto'）
```
用户修改导演风格
       ↓
   保存成功
       ↓
自动触发应用到所有资源
       ↓
（根据设置决定是否显示确认对话框）
```

## 💡 使用建议

### 推荐配置（大多数用户）

```typescript
{
  mode: 'manual',                // 手动模式（避免误操作）
  autoApplyToNew: true,          // 新建自动应用（提升效率）
  protectManualEdits: true,      // 保护编辑（防止覆盖）✨ 强烈推荐
  confirmBeforeApply: true,      // 显示确认（避免误操作）✨ 强烈推荐
  showPreview: true,             // 显示预览（查看效果）
}
```

### 高级用户配置

```typescript
{
  mode: 'auto',                  // 自动模式（快速迭代）
  autoApplyToNew: true,          // 新建自动应用
  protectManualEdits: false,     // 不保护（覆盖全部）
  confirmBeforeApply: false,     // 不确认（直接应用）
  showPreview: false,            // 不预览（更快）
}
```

### 谨慎配置（精细控制）

```typescript
{
  mode: 'manual',                // 手动模式
  autoApplyToNew: false,         // 不自动应用（完全手动）
  protectManualEdits: true,      // 保护编辑
  confirmBeforeApply: true,      // 显示确认
  showPreview: true,             // 显示预览
}
```

```

        ↓

```
用户点击"应用导演风格到全部"

- ✅ 透明度：用户知道会发生什么
