### 2. 📊 优化策略详解

#### 2.1 提示词优化策略

**结构化输出策略：**
```
【重要】请严格按照以下结构输出，不要添加任何额外说明：
...
请确保输出是有效的 JSON 格式，不要使用 Markdown 代码块。
```

**Few-shot Learning（示例学习）：**
```
【参考示例】
示例 1：
输入：小明走进房间，看到桌上有一封信。
输出：{
  "sceneNumber": 1,
  "location": "房间",
  ...
}
```

**Chain-of-Thought（分步推理）：**
```
【分析步骤】
1. 识别场景：确定地点、时间、内外景
2. 提取角色：列出所有出现的角色
3. 分析动作：将叙述转换为视觉化描述
4. 提取对白：识别并格式化所有对话
5. 估算时长：根据内容量估算场景时长
```

**自我验证策略：**
```
【输出前请自我检查】
- 检查是否所有必需字段都已填写
- 检查场景编号是否连续
- 检查角色名称是否一致
- 检查对白是否完整
- 检查时长估算是否合理
```

## 🎯 优化策略的技术原理

### 1. Few-shot Learning（少样本学习）
通过提供 1-3 个高质量示例，让 AI 快速理解期望的输出格式和质量标准。

**原理：**
- AI 通过示例学习模式
- 减少格式错误
- 提高输出一致性

### 2. Chain-of-Thought（思维链）
引导 AI 分步思考，而不是直接输出结果，提高逻辑性和准确性。

**原理：**
- 将复杂任务分解为简单步骤
- 每步都有明确目标
- 减少遗漏和错误

### 3. Self-Verification（自我验证）
让 AI 在输出前自我检查，发现并修正错误。

**原理：**
- AI 扮演检查者角色
- 主动发现问题
- 提高输出质量

### 4. Structured Output（结构化输出）
强制 AI 按照严格的格式输出，减少解析错误。

**原理：**
- 明确的格式约束
- 减少歧义
- 便于程序解析

### 5. Retry with Validation（验证重试）
每次输出都验证，失败则重试，确保最终质量。

**原理：**
- 多次尝试提高成功率
- 验证确保质量
- 指数退避避免浪费
